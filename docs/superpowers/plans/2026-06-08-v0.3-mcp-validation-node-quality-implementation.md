# v0.3 MCP Validation And Node Quality Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Improve workflow generation quality by using deeper n8n MCP workflow-builder capabilities before saving or previewing workflows.

**Architecture:** Keep the v0.2 safety model: OpenCode still creates inactive, locally managed workflow drafts through the existing REST API and registry pipeline. v0.3 adds an MCP quality layer around that pipeline: suggested-node discovery, stricter node documentation selection, planner-produced SDK validation code, and save-blocking `validate_workflow` checks. The implementation remains opt-in through existing build/update flows and does not claim/import existing workflows.

**Tech Stack:** TypeScript, Vitest, zod, n8n REST API, n8n instance-level MCP JSON-RPC, OpenCode plugin API.

---

## Current Context

v0.2 created a Docker E2E baseline and pinned the test stack to n8n `2.23.4` with instance-level MCP enabled. The plugin currently calls:

- `get_sdk_reference`
- `search_nodes`
- `get_node_types`

The current generation path is:

```text
natural language prompt
  -> MCP docs/search
  -> OpencodePlanner returns WorkflowPlan JSON
  -> compileWorkflowPlan() returns n8n workflow JSON
  -> local validateWorkflowForSave()
  -> n8n REST API create/update
```

n8n MCP docs now also expose workflow-builder tools that v0.3 should use:

- `get_suggested_nodes`: curated node recommendations for categories.
- `validate_workflow`: validates Workflow SDK code and returns errors/warnings before saving.

v0.3 should not use MCP `create_workflow_from_code` or MCP `update_workflow` for persistence yet, because that would bypass existing local registry, ownership, preview, and stale-preview controls. Keep persistence in the current REST API path.

## Scope

In scope:

- Add MCP client methods for `get_suggested_nodes` and `validate_workflow`.
- Add typed parsing for MCP validation output.
- Add planner draft schemas that include both `WorkflowPlan` and SDK validation code.
- Add node-quality context from MCP suggested nodes.
- Run MCP validation before REST API create and before update preview is saved.
- Surface MCP validation errors and warnings in tool results.
- Extend Docker E2E to cover validation and suggestion paths.
- Update Chinese README for v0.3.

Out of scope:

- Existing workflow claim/import.
- Active workflow updates.
- OAuth credential setup.
- n8n project/folder placement beyond preserving existing config fields.
- Switching persistence to MCP `create_workflow_from_code` or MCP `update_workflow`.
- Full workflow visual diff.

```ts
function parseMcpWorkflowValidationResult(text: string): McpWorkflowValidationResult {
  let value: unknown
  try {
    value = JSON.parse(extractJsonObjectText(text))
  } catch {
    throw new N8nBuilderError("n8n MCP validate_workflow returned invalid JSON.", "N8N_MCP_TOOL_ERROR", {
      toolName: "validate_workflow",
    })
  }

  if (!isRecord(value) || typeof value.valid !== "boolean") {
    throw new N8nBuilderError("n8n MCP validate_workflow returned an invalid validation result.", "N8N_MCP_TOOL_ERROR", {
      toolName: "validate_workflow",
    })
  }

  const warnings = Array.isArray(value.warnings) ? value.warnings.filter(isValidationWarning) : []
  const errors = Array.isArray(value.errors)
    ? value.errors.filter((error): error is string => typeof error === "string")
    : []

  return {
    valid: value.valid,
    ...(typeof value.nodeCount === "number" ? { nodeCount: value.nodeCount } : {}),
    warnings,
    errors,
  }
}

function extractJsonObjectText(text: string): string {
  const fenceMatch = text.match(/```(?:json)?\s*([\s\S]*?)```/i)
  const candidate = (fenceMatch?.[1] ?? text).trim()
  const firstObject = candidate.indexOf("{")
  const lastObject = candidate.lastIndexOf("}")

  if (firstObject === -1 || lastObject === -1 || lastObject < firstObject) {
    return candidate
  }

  return candidate.slice(firstObject, lastObject + 1)
}

function isValidationWarning(value: unknown): value is McpWorkflowValidationWarning {
  if (!isRecord(value) || typeof value.code !== "string" || typeof value.message !== "string") {
    return false
  }

  return (
    (value.nodeName === undefined || typeof value.nodeName === "string") &&
    (value.parameterPath === undefined || typeof value.parameterPath === "string")
  )
}
```

```ts
    categories.add("data_transformation")
  }
  if (/\b(email|slack|notify|notification|提醒|通知)\b/i.test(normalized)) {
    categories.add("notification")
  }

  return [...categories].slice(0, 4)
}
```

```ts
import { N8nBuilderError } from "./errors.js"
import type { Warning } from "./types.js"
import type { N8nWorkflow } from "./validator.js"
import type { McpWorkflowValidationResult } from "./n8n-mcp-client.js"

export type McpWorkflowValidator = {
  validateWorkflowCode(code: string): Promise<McpWorkflowValidationResult>
}

export type ValidateWorkflowWithMcpInput = {
  mcp: McpWorkflowValidator
  workflow: N8nWorkflow
  sdkCode: string
}

export async function validateWorkflowWithMcp(input: ValidateWorkflowWithMcpInput): Promise<Warning[]> {
  const result = await input.mcp.validateWorkflowCode(input.sdkCode)

  if (!result.valid) {
    throw new N8nBuilderError("n8n MCP workflow validation failed.", "MCP_WORKFLOW_VALIDATION_FAILED", {
      errors: result.errors,
      warnings: result.warnings.map(toDetailsWarning),
    })
  }

  if (typeof result.nodeCount === "number" && result.nodeCount !== input.workflow.nodes.length) {
    throw new N8nBuilderError(
      "n8n MCP workflow validation returned a different node count than the compiled workflow.",
      "MCP_WORKFLOW_VALIDATION_MISMATCH",
      {
        expectedNodeCount: input.workflow.nodes.length,
        validatedNodeCount: result.nodeCount,
      },
    )
  }

  return result.warnings.map((warning) => ({
    code: `MCP_${warning.code}`,
    message: warning.message,
    nodeName: warning.nodeName,
  }))
}

function toDetailsWarning(warning: { code: string; message: string; nodeName?: string; parameterPath?: string }) {
  return {
    code: warning.code,
    message: warning.message,
    nodeName: warning.nodeName,
    parameterPath: warning.parameterPath,
  }
}
```
