# v1.0 Release Candidate Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Freeze the public behavior for a stable `1.0.0` release candidate with documented contracts, reviewed safety posture, release-ready metadata, and no known critical workflow lifecycle blockers.

**Architecture:** Keep runtime behavior stable and avoid new workflow features. Make the public contract explicit through exported TypeScript types plus docs, then add release-candidate tests that verify package exports, docs coverage, version metadata, safety guardrail documentation, and final release checklist status.

**Tech Stack:** TypeScript, Vitest, tsup, Markdown docs, Node.js package exports, OpenCode plugin API, n8n REST API/MCP contract documentation.

- New workflow runtime features.
- Active workflow structural editing.
- Automatic OAuth consent.
- Exhaustive verified support claim for every n8n node configuration.
