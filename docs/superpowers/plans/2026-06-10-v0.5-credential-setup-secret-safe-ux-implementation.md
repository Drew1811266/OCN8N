# v0.5 Credential Setup And Secret-Safe UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Do not implement v0.6 claim/import or active workflow behavior while executing this plan.

**Goal:** Make credential requirements understandable and actionable for users while preserving the project's no-secret-leakage guarantees.

**Architecture:** Keep v0.4 workflow generation, preview/apply, registry, and MCP validation behavior unchanged. Add a credential setup action layer beside the existing `missingCredentials` result so OpenCode can explain whether a credential was reused, created from environment variables, blocked by missing environment variables, requires config mapping, or needs manual OAuth completion in n8n UI. Strengthen redaction helpers and tests for nested secret-looking data in resolver/API/MCP error details.

**Tech Stack:** TypeScript, Vitest, zod, OpenCode plugin API, n8n REST API credentials endpoint, n8n MCP JSON-RPC, existing Docker-free unit tests.

---

## Current Context

v0.4.0 has:

- `CredentialResolver` that reuses existing credentials, creates credentials from configured environment variables, or returns a `CredentialGap`.
- Tool results with `missingCredentials` but no structured next-step actions.
- `CredentialEnvMapping` with only `name`, `type`, and `env`.
- Security checks for plaintext secrets in workflow node parameters.
- API and MCP clients that avoid echoing API keys/tokens in normal errors, but no shared exported redaction utility for nested details.

v0.5 must preserve:

- No plaintext secrets in workflow JSON, registry, preview files, logs, or ordinary tool output.
- No OAuth browser automation.
- No v0.6 workflow claim/import behavior.
- No active workflow update behavior.
- Existing `missingCredentials` field for backward compatibility.

## Scope

In scope:

- Add `credentialActions` to build/update results.
- Expand credential config metadata with optional `authMode`, `setupUrl`, and `docs`.
- Return structured setup actions for:
  - reusing existing n8n credentials
  - creating credentials from environment variables
  - missing environment variables
  - missing credential mapping
  - manual OAuth setup in n8n UI
- Add helper functions for safe manual credential setup URLs derived from configured n8n base URL.
- Add shared recursive redaction helper for nested details.
- Add tests that prove no secret values are returned in credential actions, gaps, errors, or redacted details.
- Update README with practical Slack, HTTP Header Auth, generic API key, SMTP, and OAuth handoff examples.
- Bump version metadata to `0.5.0`.

Out of scope:

- OAuth consent automation.
- Credential rotation.
- Storing secrets in local config.
- Claiming/importing existing workflows.
- Production activation.

## Public Result Shape

Keep `missingCredentials` unchanged, and add `credentialActions`:

```ts
export type CredentialActionStatus = "resolved" | "required"

export type CredentialActionType =
  | "reuse_existing"
  | "create_from_env"
  | "set_missing_env"
  | "configure_mapping"
  | "complete_oauth_in_n8n"

export type CredentialSetupAction = {
  nodeName: string
  credentialType: string
  credentialName?: string
  action: CredentialActionType
  status: CredentialActionStatus
  message: string
  requiredEnv?: string[]
  manualSetupUrl?: string
  docs?: string[]
}
```

Rules:

- `status: "resolved"` means no user action is required for this credential reference.
- `status: "required"` means the workflow can be created as inactive draft, but the user must act before activation.
- `requiredEnv` contains environment variable names only, never values.
- `manualSetupUrl` must be derived from n8n base URL or explicit config; never include secrets.
- `docs` contains short public documentation labels or URLs, not secret data.

```ts
const credentialAuthModes = new Set(["api_key", "oauth2", "manual"])

function readCredentialAuthMode(field: string, value: unknown): CredentialAuthMode | undefined {
  if (value === undefined) return undefined
  if (typeof value !== "string" || !credentialAuthModes.has(value)) {
    throwInvalidConfig(field, "must be one of api_key, oauth2, manual")
  }

  return value as CredentialAuthMode
}

function readOptionalStringArray(field: string, value: unknown): string[] | undefined {
  if (value === undefined) return undefined
  if (!Array.isArray(value) || !value.every((item) => typeof item === "string")) {
    throwInvalidConfig(field, "must be an array of strings")
  }

  return value
}
```

Update `readCredentialEnvMapping` return value:

```ts
return {
  name: value.name,
  type: value.type,
  env,
  ...(readCredentialAuthMode(`${field}.authMode`, value.authMode) ? {
    authMode: readCredentialAuthMode(`${field}.authMode`, value.authMode),
  } : {}),
  ...(typeof value.setupUrl === "string" ? { setupUrl: value.setupUrl } : {}),
  ...(readOptionalStringArray(`${field}.docs`, value.docs) ? {
    docs: readOptionalStringArray(`${field}.docs`, value.docs),
  } : {}),
}
```

export function credentialSetupUrl(baseUrl: string): string {
  const appBaseUrl = baseUrl.replace(/\/api\/v\d+\/?$/i, "").replace(/\/+$/, "")

}

const redactedValue = "[REDACTED]"

export function redactSecrets(value: unknown): unknown {
  return redactValue(value, undefined)
}

function redactValue(value: unknown, key: string | undefined): unknown {
  if (isSecretKey(key ?? "")) {
    return redactedValue
  }

  if (typeof value === "string") {
    return isSecretString(value) ? redactedValue : value
  }

  if (Array.isArray(value)) {
    return value.map((item) => redactValue(item, undefined))
  }

  if (value && typeof value === "object") {
    return Object.fromEntries(
      Object.entries(value as Record<string, unknown>).map(([childKey, childValue]) => [
        childKey,
        redactValue(childValue, childKey),
      ]),
    )
  }

  return value
}

function isSecretString(value: string): boolean {
  return /\bBearer\s+[A-Za-z0-9._~+/=-]+/i.test(value) || /\bxox[baprs]-[A-Za-z0-9-]+/i.test(value)
}
